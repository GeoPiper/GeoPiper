GeoPiper

Meeting Data Where It Lives

Finding, accessing, and subsettting earth system data is one of the largest bottle necks to research and operations. GeoPiper seeks to eliminate the need to download, store and format data by "meeting data where it lives", to provide a streamline, language agnostic way to get the data you need to solve the problems you have.

What is the GeoPiper Framework?

Federated Data Catalog

GeoPiper collects, validates, and harmomizes a catalog of information describing where open data lives, how it is structured, and the spatial, temporal, and thematic domains it covers. We explictly avoid deeply nested list structures for data dissemination to make it easier to find, ingest, and share source data.

Language-agnostic Framework

The catalog on its own is a key GeoPiper products, but in addtion we supply a variety of software for accessing data in the catalog using R and Python.

In addition to data access libraries we provide a number of open source libraries for summarizing and aggregating data products to answer a range of multiscalar problems in a consistent platform.

How does it work?

Flat Data Catalog

We aim to buildo nested lists, stores mimiaml information to effectivly describe the spatial, temporal and theamic information of a data set. Set to crawl over thousands of identifed sources of high quality to document hundreds of thousands web acccessable data products and the approriate access protocols

GeoPiper provides three a flat data catalog of federated data sources that are web acccessable via JSON and Parquet files through the public Lynker s3 bucket. Additonally, it provides a set of tools to extract, data from a set remote resource into an in memory object.

Automatic Updates & Growing

Maintaining a collection of hundreds of thousands of resources is not manually feasable. To overcome this we have identifed a set of critical data and methods for automatically describing resources as catalog elements.

We have implemented CI/CD routines that run monthly to ensure resources are avaiable and havent changed and developed.vValidators are used to ensure that accurate data is returned.

Search & Query

A challenge with so many possible resources is the abilty to search a find the approariate data for a given problem. Our tools provide a (currently) primative NLP process to discover viable dataset for your needs.

Programmatic Extraction

We support both R and Python packages to extract data from catalog elements for provided spatial and temporal bounds.
